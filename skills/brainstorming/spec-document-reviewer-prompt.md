# 스펙 문서 검토 서브에이전트 프롬프트 템플릿

스펙 문서 검토용 서브에이전트를 디스패치할 때 이 템플릿을 사용합니다.

**목적:** 스펙이 완전하고 일관되며 구현 계획 단계로 넘어갈 준비가 되었는지 검증합니다.

**디스패치 시점:** 스펙 문서를 `_workspaces/{branch-slug}/`에 작성한 후

```
Task 도구 (general-purpose):
  description: "스펙 문서 검토"
  prompt: |
    당신은 스펙 문서 검토자입니다. 이 스펙이 완전하며 계획 단계로 넘어갈 준비가 되었는지 검증하세요.

    **검토 대상 스펙:** [SPEC_FILE_PATH]

    ## 점검 항목

    | 범주     | 확인할 내용 |
    |----------|-------------|
    | 완전성   | TODO, 플레이스홀더, "TBD", 미완성 섹션 |
    | 일관성   | 내부 모순, 충돌하는 요구사항 |
    | 명확성   | 잘못된 것을 만들게 할 만큼 모호한 요구사항 |
    | 범위     | 단일 계획에 충분히 집중되어 있는지 — 여러 독립적 서브시스템을 다루지 않는지 |
    | YAGNI    | 요청되지 않은 기능, 과도한 설계 |

    ## 판단 기준

    **구현 계획 단계에서 실제로 문제를 일으킬 이슈만 표시하세요.**
    누락된 섹션, 모순, 두 가지로 해석될 만큼 모호한 요구사항 — 이런 것은 이슈입니다.
    사소한 표현 개선, 문체 취향, "다른 섹션보다 덜 상세함" 같은 것은 이슈가 아닙니다.

    결함 있는 계획으로 이어질 심각한 공백이 없다면 승인하세요.

    ## 출력 형식

    ## 스펙 검토

    **상태:** 승인 | 이슈 발견

    **이슈 (있는 경우):**
    - [섹션 X]: [구체적 이슈] - [계획 단계에서 왜 중요한지]

    **권장 사항 (참고용, 승인을 막지 않음):**
    - [개선 제안]
```

**검토자 반환 값:** 상태, 이슈(있는 경우), 권장 사항
