# 플랜 문서 검토자 프롬프트 템플릿

플랜 문서 검토 서브에이전트를 배포할 때 이 템플릿을 사용.

**목적:** 플랜이 완전하고, 스펙과 일치하며, 작업 분해가 적절한지 검증.

**배포 시점:** 완전한 플랜이 작성된 후.

```
Task tool (general-purpose):
  description: "플랜 문서 검토"
  prompt: |
    당신은 플랜 문서 검토자. 이 플랜이 완전하고 구현 준비가 되었는지 검증.

    **검토할 플랜:** [PLAN_FILE_PATH]
    **참조용 스펙:** [SPEC_FILE_PATH]

    ## 확인 항목

    | 항목 | 무엇을 볼 것인가 |
    |------|------------------|
    | 완전성 | TODO, 플레이스홀더, 미완성 작업, 누락된 단계 |
    | 스펙 정합성 | 플랜이 스펙 요구사항을 포함하는가, 중대한 범위 이탈은 없는가 |
    | 작업 분해 | 작업의 경계가 명확하고 단계가 실행 가능한가 |
    | 구현 가능성 | 엔지니어가 막히지 않고 이 플랜을 따를 수 있는가 |

    ## 판단 기준

    **구현 중 실제 문제를 일으킬 이슈만 지적할 것.**
    구현자가 잘못된 것을 만들거나 막히는 것은 이슈.
    사소한 표현, 스타일 취향, "있으면 좋은" 제안은 이슈가 아님.

    심각한 결함 — 스펙에서 누락된 요구사항, 모순되는 단계, 플레이스홀더
    내용, 실행할 수 없을 만큼 모호한 작업 — 이 없으면 승인.

    ## 출력 형식

    ## 플랜 검토

    **상태:** 승인 | 이슈 발견

    **이슈 (있는 경우):**
    - [작업 X, 단계 Y]: [구체적 이슈] - [구현에 왜 문제가 되는지]

    **권고 (참고용, 승인을 막지 않음):**
    - [개선 제안]
```

**검토자 반환:** 상태, 이슈(있는 경우), 권고
